# 화림 업무전산시스템 배포 메모

## 권장 구조

이 앱은 세 부분으로 나누어 운영하는 방식이 가장 안정적입니다.

- GitHub Pages: 직원들이 접속하는 앱 주소
- Google Sheets: 업무, 자재, 최종현황, 앱 백업 데이터 저장소
- Google Apps Script: 앱과 구글시트를 연결하는 서버

로컬 `file:///.../index.html`은 테스트용입니다. 여러 사람이 쓰려면 GitHub Pages 같은 공개 웹주소로 배포해야 합니다.

## 구글시트 준비

1. 프로젝트별 구글시트를 만듭니다.
2. 각 시트에서 `확장 프로그램 > Apps Script`를 엽니다.
3. `google-apps-script.gs` 내용을 붙여 넣습니다.
4. `setupSheets` 함수를 한 번 실행합니다.
5. `배포 > 새 배포 > 웹 앱`으로 배포합니다.
6. 실행 권한은 본인, 접근 권한은 앱 사용자 정책에 맞게 설정합니다.
7. 발급된 `/exec` URL을 앱의 `환경설정`에 저장하거나 `app-config.js`에 넣습니다.

## GitHub Pages 배포

1. GitHub에 새 저장소를 만듭니다.
2. 이 폴더의 파일을 저장소에 올립니다.
3. `app-config.js`에서 프로젝트별 Apps Script URL을 넣습니다.
4. 저장소의 `Settings > Pages`에서 배포 브랜치를 선택합니다.
5. 발급된 GitHub Pages 주소를 현장 직원들에게 공유합니다.

## 데이터 저장 범위

구글시트에는 다음 데이터가 저장됩니다.

- `업무현황`: 업무등록 입력값
- `자재현황`: 자재등록 입력값
- `최종현황`: 대시보드 요약 스냅샷과 추가항목 금액
- `앱저장소`: 환경설정, 최종현황 추가항목, 복원용 백업

다른 PC에서 사용할 때는 앱의 `환경설정 > 구글시트에서 불러오기`로 구글시트 백업을 불러올 수 있습니다.
